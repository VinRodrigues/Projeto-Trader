Ordem de Compra

UMLUseCase
Projeto 1 - Diagrama de caso de uso :: Model :: Ordem de Compra

Description

+Fluxo Principal

  1. O usuário acessa a interface de ativos[FS001]
  2. O usuário verifica valores de ativos.
  3. O usuário seleciona o ativo que deseja comprar.
  4. O usuário visualiza um gráfico infraday e histórico da ação, o gráfico mostra o valor atual, abertura, fechamento anterior, máximo e mínimo.
  5. O usuário envia ordens de compra.[FS002]
  6. B3 avalia ordem de compra .
  7. B3 autoriza a ordem de compra.

+Fluxo Secundário[FS001]

  1. Usuário visualiza ativos comprados.
  2. Usuário checa o status da ordem de compra.
  3. Usuário visualiza OC relacionada a ação relacionada.

+Fluxo Secundário[FS002]

  1. Usuário solicita OC por quantidade de ações ou por valor financeiro.
  2. Usuário checa o status da ordem de compra.
